PERFORMANCE REQUIREMENTS:
   Strengths:
      Right areas identified (responsiveness, network failures, image
         loading).
   Suggestions:
      Current PRs mix security/UX statements with performance (e.g.,
         “accessible only to me”, “respond instantly”, “notify on API
         failure”, “load assets efficiently”). Only one is latency-like and
         still qualitative (“instantly”).
      Replace qualitative phrasing with quantitative budgets, e.g.:
         – Dashboard initial load =2.0 s at p95 on broadband; =4.0 s on 3G.

            trip) at p95.

         – Background analytics recompute =10 s for N=10k items.
         – Degradation rule when external APIs fail (fallback in =200 ms).
